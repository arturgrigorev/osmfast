osmfast poi

Extract points of interest by category

← Back to Documentation
Description

The poi command extracts points of interest from OSM data, organized into logical categories: - food: restaurants, cafes, bars, fast food - shop: retail stores of all types - health: hospitals, clinics, pharmacies - education: schools, universities, libraries - transport: bus stops, stations, parking - tourism: hotels, museums, attractions - finance: banks, ATMs - service: post offices, police, fire stations - worship: churches, mosques, temples, synagogues

Usage
osmfast poi map.osm -o pois.geojson
osmfast poi --category food map.osm -o food.geojson
osmfast poi --list-categories
osmfast poi --named-only map.osm -o named_pois.geojson
Examples
All POIs
osmfast poi city.osm -o pois.geojson
Restaurants
osmfast poi --category food city.osm -o food.geojson
Shops
osmfast poi --category shop city.osm -o shops.geojson
List categories
osmfast poi --list-categories
Options
Option Description
-o, --output Output file (default: stdout)
-f, --format Output format: geojson, json, csv
-c, --category POI category: food, shop, health, education, transport, tourism, finance, service, worship, all
--include-ways Include POIs from ways (use centroid)
--named-only Only POIs with names
--stats Show statistics only
--list-categories List available categories and exit
Output

GeoJSON with properties: - id: OSM node/way ID - category: POI category - type: Specific amenity/shop type - name: POI name - address: Full address if available - phone: Contact phone if tagged - website: Website URL if tagged - opening_hours: Hours if tagged

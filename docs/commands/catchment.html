osmfast catchment

Generate catchment areas with POI analysis

← Back to Documentation
Description

The catchment command creates travel time zones showing what's reachable within specified times. It generates: - Isochrone polygons for multiple travel times - POI analysis within each zone - Categorized amenity counts Useful for location analysis, site selection, and accessibility studies.

Usage
osmfast catchment map.osm --lat 51.5 --lon -0.1 -o catchment.geojson
osmfast catchment map.osm --lat 51.5 --lon -0.1 --time 5,10,15,20 --mode walk
osmfast catchment map.osm --lat 51.5 --lon -0.1 --categories food,healthcare
Examples
Basic catchment
osmfast catchment city.osm --lat 51.5 --lon -0.1 -o catchment.geojson
Custom times
osmfast catchment city.osm --lat 51.5 --lon -0.1 --time 5,10,20,30 -o catchment.geojson
Cycling mode
osmfast catchment city.osm --lat 51.5 --lon -0.1 --mode bike -o bike_catchment.geojson
Food and healthcare
osmfast catchment city.osm --lat 51.5 --lon -0.1 --categories food,healthcare -o catchment.geojson
Driving catchment
osmfast catchment city.osm --lat 51.5 --lon -0.1 --mode drive --time 5,10,15 -o drive.geojson
Options
Option Description
--lat Origin latitude (required)
--lon Origin longitude (required)
-t, --time Travel times in minutes, comma-separated (default: 5,10,15)
-m, --mode Travel mode: walk, bike, drive (default: walk)
-c, --categories POI categories to analyze (comma-separated)
-f, --format Output format: json, geojson, text (default: geojson)
--resolution Polygon boundary resolution (default: 36)
-o, --output Output file (required)
Output

Catchment analysis with: - Isochrone polygon for each travel time - POI counts by category (healthcare, food, education, etc.) - Individual POI locations with travel times - Origin point marker

osmfast sort

Sort features by attribute

← Back to Documentation
Description

The sort command sorts OSM features by a tag value or property. Supports: - Sorting by any tag (name, population, elevation, etc.) - Sorting by built-in properties (id, lat, lon, type) - Numeric sorting for numeric values - Ascending or descending order - Output limiting Useful for finding extremes or creating ordered lists.

Usage
osmfast sort map.osm --by name -o sorted.geojson
osmfast sort map.osm --by population --numeric --reverse -n 100
osmfast sort map.osm --filter highway=* --by maxspeed --numeric
Examples
Sort by name
osmfast sort city.osm --by name -o sorted.geojson
Top 100 by population
osmfast sort city.osm --filter place=* --by population --numeric --reverse -n 100 -o top_places.geojson
Highest elevations
osmfast sort region.osm --filter natural=peak --by ele --numeric --reverse -n 50 -o highest.geojson
Roads by speed limit
osmfast sort city.osm --filter highway=* --by maxspeed --numeric --reverse -o fast_roads.geojson
Sort by latitude
osmfast sort city.osm --by lat -o sorted.geojson
CSV output
osmfast sort city.osm --by name -f csv -o sorted.csv
Options
Option Description
-b, --by Sort by: tag name, 'id', 'lat', 'lon', 'type' (required)
-r, --reverse Sort in descending order
--numeric Sort numerically (for numeric tag values)
--filter Filter by tag before sorting (e.g., highway=*)
-n, --limit Limit output to N elements
-f, --format Output format: geojson, json, csv (default: geojson)
-o, --output Output file (required)
Output

Sorted features in the specified format, with the sort key included in CSV output.

osmfast bottleneck

Identify network chokepoints and critical links

← Back to Documentation
Description

The bottleneck command identifies critical points in the road network where disruption would have the greatest impact on connectivity. It finds: - Bridge edges (edges whose removal disconnects the graph) - Articulation points (nodes whose removal disconnects the graph) - High-betweenness edges (frequently used in shortest paths) Useful for infrastructure planning and resilience analysis.

Usage
osmfast bottleneck map.osm
osmfast bottleneck map.osm --top 20
osmfast bottleneck map.osm -f geojson -o bottlenecks.geojson
Examples
Find bottlenecks
osmfast bottleneck city.osm
Top 20 chokepoints
osmfast bottleneck city.osm --top 20
Top 50 chokepoints
osmfast bottleneck city.osm -n 50
Export as GeoJSON
osmfast bottleneck city.osm -f geojson -o bottlenecks.geojson
JSON output
osmfast bottleneck city.osm -f json
Options
Option Description
-n, --top Number of top bottlenecks to return (default: 20)
-o, --output Output file (default: stdout)
-f, --format Output format: text, json, geojson
Output

List of network bottlenecks with: - Type (bridge edge, articulation point, high betweenness) - Location coordinates - Connected road names - Impact score - Components affected if removed
